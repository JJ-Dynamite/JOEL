# Roadmap

JOEL's development roadmap outlines the planned features and improvements across multiple phases.

## Current Status

**Version**: 0.1.0  
**Current Phase**: Phase 2 (Compilation & Types) ✅

## Phase 1: Core Language ✅

**Status**: Complete

- ✅ Lexer - Tokenizes JOEL source code
- ✅ Parser - Builds AST from tokens
- ✅ VM/Interpreter - Executes `[Interpreted]` mode
- ✅ Basic syntax support (variables, functions, control flow)
- ✅ Built-in functions (`print`, `range`)
- ✅ Documentation site with Nextra
- ✅ CLI tool (`joel` command)
- ✅ Example programs

## Phase 2: Compilation & Types ✅

**Status**: Complete

- ✅ LLVM backend infrastructure for `[Compiled]` mode
- ✅ WASM target compilation infrastructure
- ✅ Static type checking
- ✅ Type inference improvements
- ✅ Ownership system (Rust-like borrow checker)
- ✅ Error messages and diagnostics
- ✅ Standard library core modules

**Documentation**: 
- [Type System Guide](/guides/type-system)
- [Ownership System](/guides/ownership)

## Phase 3: Specialized Targets ✅

**Status**: Complete

### Blockchain Targets
- [x] EVM bytecode generation for smart contracts
- [x] Solana WASM target (BPF generation)
- [x] Cosmos SDK target (WASM contracts)
- [x] Polkadot/Substrate target (WASM runtime modules)

### Native & Web Targets
- [x] LLVM IR generation
- [x] WASM binary generation
- [x] Native optimization support (--optimize flag)
- [x] Dead code elimination
- [x] Constant folding
- [x] Inlining hints
- [x] Cross-compilation support (--arch flag)
- [x] Debug symbols and source maps (--debug flag)

**Documentation**: [CLI Reference](/toolchain/cli)

### Mobile Targets
- [x] iOS target (LLVM IR for iOS frameworks)
- [x] Android target (LLVM IR for Android libraries)
- [ ] React Native integration
