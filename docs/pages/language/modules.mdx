# Modules

Modules help organize code into reusable units.

## Module Declaration

```joel
module my_module

# Module code here
```

## Importing Modules

### Basic Import

```joel
import std
import math
```

### Import with Alias

```joel
import ai as ml
import chain as web3
```

### Import Specific Items

```joel
# Coming soon
import { add, subtract } from math
```

## Module Structure

```joel
module calculator

fn add(a: i32, b: i32) -> i32 {
  return a + b
}

fn subtract(a: i32, b: i32) -> i32 {
  return a - b
}

fn multiply(a: i32, b: i32) -> i32 {
  return a * b
}
```

## Using Imported Modules

```joel
import calculator

fn main() {
  let result = calculator.add(5, 3)
  print(result)  # 8
}

main()
```

## Examples

### Creating a Module

```joel
# math.joel
module math

fn add(a: i32, b: i32) -> i32 {
  return a + b
}

fn subtract(a: i32, b: i32) -> i32 {
  return a - b
}
```

### Using a Module

```joel
# main.joel
import math

fn main() {
  print(math.add(10, 5))      # 15
  print(math.subtract(10, 5)) # 5
}

main()
```

## Best Practices

1. **One module per file**: Keep modules focused
2. **Clear naming**: Use descriptive module names
3. **Organize by feature**: Group related functionality
4. **Avoid circular imports**: Keep dependencies acyclic
5. **Document exports**: Comment public APIs
